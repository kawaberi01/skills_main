## 必須チェック項目

### A. アーキテクチャ・設計
- [ ] 既存のアーキテクチャパターンに準拠しているか
- [ ] SOLID原則に従っているか
- [ ] 適切な責任分離がされているか
- [ ] 依存関係が適切に管理されているか

### B. コード品質
- [ ] 命名規則が統一されているか（C#/.NET規約準拠）
- [ ] コードの可読性が十分か
- [ ] 重複コードがないか
- [ ] 不要なコメントや古いコメントがないか
- [ ] マジックナンバーの使用がないか

### C. 機能性・ロジック
- [ ] 要件を正しく実装しているか
- [ ] エッジケースが考慮されているか
- [ ] エラーハンドリングが適切か
- [ ] 入力値の検証が十分か
- [ ] 戻り値の型と内容が適切か

### D. パフォーマンス
- [ ] 不要なデータベースクエリがないか
- [ ] N+1問題の可能性はないか
- [ ] メモリリークの可能性はないか
- [ ] 非効率なアルゴリズムの使用がないか
- [ ] 適切な非同期処理が実装されているか

### E. セキュリティ
- [ ] SQLインジェクションの脆弱性はないか
- [ ] XSS攻撃への対策がされているか
- [ ] 認証・認可の実装が適切か
- [ ] 機密情報のハードコードがないか
- [ ] 入力値サニタイゼーションが実装されているか

### F. テスト
- [ ] 単体テストが適切にカバーされているか
- [ ] テストケースが網羅的か
- [ ] テストが実際に機能を検証しているか
- [ ] モックの使用が適切か

### G. 既存コードとの整合性
- [ ] プロジェクトの既存パターンに従っているか
- [ ] 既存のヘルパー・ユーティリティを適切に使用しているか
- [ ] 既存APIとの互換性が保たれているか
- [ ] データベーススキーマとの整合性が取れているか

## レビュー結果出力フォーマット

以下の形式でレビュー結果を出力する：

```markdown
# コードレビュー結果

## PR概要
- **PR ID**: [PR番号]
- **タイトル**: [PRタイトル]
- **変更ファイル数**: [ファイル数]
- **関連Issue**: [Redmineリンクがあれば記載]

## 変更ファイル詳細レビュー
### [ファイル名1]
**変更内容**: [変更の概要]
**評価**: 🔴重大 / 🟡軽微 / 🟢良好
**指摘事項**:
- [具体的な指摘]

### [ファイル名2]
...

## 総合評価
**全体評価**: 🔴重大な問題あり / 🟡軽微な問題あり / 🟢承認推奨

### 重大な問題 (🔴)
- [問題の詳細とリスク]

### 軽微な問題 (🟡)
- [改善提案]

### 良好な点 (🟢)
- [評価できる実装]

## 推奨アクション
- [ ] [修正が必要な項目]
- [ ] [改善提案]

## 最終判定
**マージ可否**: ✅承認 / ❌要修正 / ⚠️条件付き承認
**理由**: [判定理由]
```
