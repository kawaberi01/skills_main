# 予想モード

予想のぶれを減らすため、モードを固定して使うこと。

## 総合買い目型

- internal key: `integrated_betting`
- 用途:
  - 実際に買う前提で、最も実務的な総合予想を出す
- 重視するデータ:
  - 勝ちパターン分析
  - 当日オッズ
  - 馬体重と増減
  - 当日馬場状態
- 狙い:
  - 軸馬と相手を決め、無理のない買い目に落とす
- 基本動作:
  - パターン上位と市場支持が重なる馬を主軸にする
  - 馬場状態が取れる場合は、`pattern_uma.track_condition_rates` の該当列を通常時より強めに見る
  - `GET /nankan/leading/jockeys` は短距離戦や接戦時の順位補正に使う
  - リーディングジョッキーは `pattern_kis` / `pattern_kis_cho` と同方向のときだけ加点を強める
  - 大穴は、パターンとオッズの両方に根拠があるときだけ拾う

## 勝ちパターン重視型

- internal key: `pattern_first`
- 用途:
  - 勝ちパターン分析そのものを中心に見たいとき
- 重視するデータ:
  - 勝ちパターン分析を最優先
  - オッズと馬体重は補足
  - 馬場状態は条件一致の確認に使う
- 狙い:
  - 騎手、馬、調教師、騎手×調教師の列から上位馬を説明する
- 基本動作:
  - サンプル数が少ない高率は過信しない
  - 条件一致の列を優先して理由を書く
  - 馬場状態がある場合は、該当する `track_condition_rates` を条件一致列として優先する
  - リーディングジョッキーは勝ちパターン分析の補助とし、騎手列の裏取りにだけ使う

## 堅実型

- internal key: `conservative`
- 用途:
  - 的中率を優先したいとき
- 重視するデータ:
  - オッズ
  - 勝ちパターン分析
  - 馬体重
  - 当日馬場状態
- 狙い:
  - 配当よりも取りやすさを優先する
- 基本動作:
  - 極端な三連系を避ける
  - 馬場悪化時は、馬場適性が見える馬を相手本線に残す
  - 複勝、ワイド、馬連寄りにまとめる

## 妙味重視型

- internal key: `value_focus`
- 用途:
  - 市場が過小評価している馬を探したいとき
- 重視するデータ:
  - 勝ちパターン分析
  - オッズ
  - 馬体重
  - 当日馬場状態
- 狙い:
  - パターン評価に対して売れすぎていない馬を拾う
- 基本動作:
  - 人気馬の過剰評価を避ける
  - 馬場適性があるのに売れていない馬を拾う
  - 単勝、馬連、馬単、選択的な三連複に寄せる

## 攻め型

- internal key: `aggressive_combo`
- 用途:
  - 三連系や高配当を狙いたいとき
- 重視するデータ:
  - 勝ちパターン分析
  - オッズ
  - 馬体重
  - 当日馬場状態
- 狙い:
  - 軸を定めて高配当の組み合わせを絞る
- 基本動作:
  - それでも軸馬の妥当性は必要
  - 馬場適性の偏りが強いときは、その条件に寄せて相手を絞る
  - 点数を広げすぎず、形を明確にする

## 共通の見方

基本の優先順は次の通りです。

1. 馬、騎手、調教師、騎手×調教師の重なり
2. 当日オッズの支持
3. 当日馬場状態とその適性
4. 馬体重と増減
5. 当日開催傾向による小幅補正
6. リーディングジョッキーによる短距離・接戦時の小幅補正

## 取得の基本形

事前予想では、個別 API を順に取りに行く前に `prediction-summary` を優先すること。

- 既定 endpoint:
  - `GET /nankan/meetings/{date}/{course}/races/{race_no}/prediction-summary?meeting_no=...&meeting_day=...`
- 開催中で鮮度が重要なとき:
  - `refresh=true` を付ける
- summary からまず使う項目:
  - `runners`
  - `trend`
  - `leading_jockeys`
  - `meta`
- 個別 API を追加で使う場面:
  - 脚質傾向が必要
  - 詳細オッズを追加確認したい
  - summary の一部が欠けている
  - summary では足りず `prediction-bundle` の原データ確認が必要

`trend` は summary に入っていても、`usable=true` のときだけ採用する。  
`leading_jockeys` は summary 内ではレース条件に合わせて自動解決された補助材料として扱う。

勝ちパターン分析で理由を書くときは、次の列を優先すること。

- `pattern_kis.rates.kawasaki`
- `pattern_kis.rates.short` または `medium`
- `pattern_uma.rates.lifetime`
- `pattern_uma.rates.short` または `medium`
- `pattern_uma.track_condition_rates.*`
- `pattern_uma.season_rates.*`
- `pattern_uma.frame_group_rates.*`
- `pattern_cho.rates.kawasaki`
- `pattern_cho.rates.short` または `medium`
- `pattern_kis_cho.rates.kawasaki`
- `pattern_kis_cho.rates.short` または `medium`

リーディングジョッキーはこの優先順を上書きしない。`pattern_kis` / `pattern_kis_cho` と同方向のときだけ補助加点し、馬の場・距離率が弱い馬を騎手成績だけで頭候補にしない。

リーディングジョッキー補正の目安:

- 取得API:
  - `GET /nankan/leading/jockeys?course={course}&distance={distance}&track_condition={track_condition}&period=recent_3months&sort=win_rate`
- 強めに見る場面:
  - 900m から 1500m までの短距離
  - 人気上位同士の接戦
  - `pattern_kis` と `pattern_kis_cho` が同方向
- 弱めに見る場面:
  - 2歳戦
  - 馬の `pattern_uma.rates.kawasaki` と距離帯率が弱い
  - リーディング上位でも当該馬の条件適性が伴わない
- 根拠欄の書き方:
  - `騎手の川崎1400m適性が上位`
  - `当該馬場条件での騎手成績が安定`
  - `短距離戦なので騎手補正を加点`

## 頭候補・軸候補・相手候補の分け方

### 頭候補

頭候補は、次の重なりがある馬を優先すること。

- パターン総合で上位
- 当日オッズでも上位支持
- 当日馬場と `pattern_uma.track_condition_rates` が噛み合う
- 馬体重増減に大きな不安がない

追加原則:

- 馬場率だけで頭候補を決めない
- `pattern_uma.rates.kawasaki` と `pattern_uma.rates.short/medium/long` が伴う馬を頭候補に置く
- `track_condition_rates` が強くても、場・距離率の裏付けが薄ければ頭候補ではなく軸候補や相手候補に落とす
- 当日馬場に対応する `pattern_uma.track_condition_rates` が `0.0` の馬は、原則として頭候補に置かない
- `track_condition_rates = 0.0` でも頭候補へ残すなら、場率・距離率・オッズ支持・時計系の複数裏付けを明記する

頭候補にしにくい例:

- オッズ上位でも馬場適性の裏付けが弱い
- パターンは強いが相手関係や時計補正だけで押し上がっている
- 人気上位同士が接戦で抜けが弱い
- 馬場率だけが高く、場・距離率が弱い

### 軸候補

軸候補は、1着固定よりも3着内の安定感を優先して置くこと。

見る項目:

- `pattern_uma.track_condition_rates.*`
- `pattern_uma.rates.kawasaki`
- `pattern_uma.rates.short/medium/long`
- `pattern_kis_cho.rates.kawasaki`
- 当日オッズの支持

軸候補の考え方:

- 勝ち切り不安があっても、条件一致が強ければ軸に残す
- 1番人気は頭からは疑っても、軸からは簡単に外しすぎない

### 相手候補

相手候補は「総合点上位」ではなく、「残る理由がある馬」を拾うこと。

優先順:

1. `pattern_uma.track_condition_rates` 一致
2. `pattern_uma.rates.kawasaki`
3. `pattern_uma.rates.short/medium/long`
4. `pattern_kis_cho.rates.kawasaki`
5. `pattern_cho.rates.kawasaki`

追加原則:

- `pattern_uma.track_condition_rates` は相手候補の拾い上げで強く使う
- 頭候補に届かない馬でも、馬場率が高ければ相手候補へ残す
- 短距離では `pattern_kis.rates.kawasaki` と `pattern_kis_cho.rates.kawasaki` の強い馬を相手候補へ上げてよい
- 相手候補から本線へ昇格する条件を固定して扱う
- 昇格条件の目安:
  - 当日馬場率がプラス
  - 場率または距離率がプラス
  - 当日傾向や脚質想定に合う
  - 単勝オッズに妙味がある
  - 騎手または騎手×厩舎の後押しがある
- 本命が明確なレースでは、穴側の相手候補を1頭は本線へ繰り上げることを検討する

相手候補として扱いやすい型:

- `馬場相手`
  - 馬場適性で浮上する馬
- `場・距離相手`
  - 場と距離帯で崩れにくい馬
- `人の相手`
  - 騎手×厩舎、厩舎で条件一致がある馬

相手候補として弱い型:

- `lifetime` だけ高い馬
- サンプル極小の高率だけに依存する馬
- 当日馬場と逆の傾向を持つ馬
- オッズだけ売れていて条件一致が薄い馬

## 券種の切り替え方

### 複勝向き

- 軸候補は見える
- ただし頭候補の抜けが弱い
- 相手も広くなりやすい

### ワイド向き

- 軸候補の3着内期待がある
- 相手候補を2頭前後に絞れる
- 頭の自信はそこまで高くない

### 馬連 / 馬単向き

- 頭候補と相手候補の序列がある程度見える
- 人気上位接戦でも、2頭に寄せられる

### 3連複向き

- 上位3頭から4頭に収束する
- 1着固定より、3頭内に来る顔ぶれを当てに行くほうが自然

### 3連単向き

- 頭候補の理由が明確
- 軸候補も崩れにくい
- 相手候補が絞れる
- 人気上位が接戦すぎない

## レース判定

判定は次の 3 段階で出すこと。

- `実行`
  - 頭候補または軸候補に一定の裏付けがあり、本線で買ってよい
- `軽く遊ぶ`
  - 強く張るほどではないが、少額・少点数なら参加できる
- `見送り`
  - 材料不足または勝ち筋が薄く、買わないほうが自然

## 本命信頼度

本命信頼度は、買い方に落とす前に必ず `高 / 中 / 低` の3段階で置くこと。

- `高`
  - 場率、距離率、当日馬場率、オッズ支持が大きく矛盾しない
  - 相手候補も絞りやすい
- `中`
  - 軸にはできるが、馬場 `0.0`、相手分散、馬体重不安などの弱点が残る
- `低`
  - 人気先行
  - 条件の裏付けが薄い
  - 頭候補より軸候補や相手候補としての色が強い

本命信頼度の使い方:

- `高` なら `実行` を検討してよい
- `中` なら `軽く遊ぶ` 寄りで考える
- `低` なら無理に頭固定せず、見送り寄りか相手型へ落とす

## 見送り判定

次のようなレースは見送り寄りと判断すること。

- 勝ち馬候補の裏付けが薄い
- `track_condition_rates` はあるが、場・距離率が伴わない
- 人気上位が接戦で、頭候補の抜けが作れない
- パターン上は相手候補までは見えても、頭候補が決め切れない

## 低配当判定

低配当判定は、印の後ではなく買い方へ落とす直前に行うこと。

- 上位2頭でオッズが極端に低いときは、`的中寄りだが回収薄い` と明記する
- 本線ワイドや馬連が低すぎるときは、厚張りを避ける
- 穴相手を混ぜても配当妙味がほとんど出ないときは、`軽く遊ぶ` か `見送り` に落とす
- 低配当判定は、印の正しさを否定するものではなく、買う価値の判定として分けて扱う

例:

- 2026-07-06 川崎 7R のように、勝ち馬のパターン裏付けが薄いレース

## 軽く遊ぶ判定

次のようなレースは `軽く遊ぶ` に落とすこと。

- 頭候補は弱いが、軸候補はある
- 相手候補までは見える
- 人気上位接戦で、頭固定は危ない
- `track_condition_rates` は効いていそうだが、勝ち切りの場・距離率が弱い
- 見送りにするほど材料不足ではない

## 軽く遊ぶなら

`軽く遊ぶ` 判定のときは次の方針に寄せること。

- 強気の3連単より、複勝・ワイド・馬連寄りに落とす
- `頭候補` ではなく `軸候補` を中心にする
- `track_condition_rates` が高い馬を相手本線に置く
- 点数は広げすぎず、控えめにまとめる
- 出力時は `軽く遊ぶなら` と明記する

### 1番人気を2,3着側で使う条件

次のようなときは、1番人気を頭ではなく軸や相手側で扱ってよい。

- 市場支持は強いが、馬場適性の裏付けが薄い
- 条件一致では別馬のほうが上
- 1番人気は残りそうだが、勝ち切りの根拠が弱い
- 別馬に `track_condition_rates` とオッズ以外の条件一致が重なる

## 馬場状態の使い方

card API の `track_condition` を基準に、勝ちパターン分析の次の列へ対応付けること。

- `good` -> `pattern_uma.track_condition_rates.good`
- `slightly_heavy` -> `pattern_uma.track_condition_rates.slightly_heavy`
- `heavy` -> `pattern_uma.track_condition_rates.heavy`
- `bad` -> `pattern_uma.track_condition_rates.bad`

馬場状態が取得できる場合:

- 該当列が高く、かつサンプル数が極端に少なくなければ加点する
- `pattern_uma.rates.kawasaki` や `pattern_uma.rates.short/medium/long` より、該当する `track_condition_rates` を一段強く見る
- 馬場が `heavy` や `bad` のときほど、この補正を強める
- 該当列が 0.0 で、他の条件も強くない馬は下げる
- オッズが売れていなくても馬場適性がはっきりしていれば穴候補に残す

馬場状態が取得できない場合:

- 馬場補正なしとして扱い、その旨を注意点に書く

## 2歳戦での扱い

2歳戦は、持ち時計と上がり時計の信頼度を通常戦より下げて扱うこと。

原則:

- `best-time` と `closing-speed` だけで順位を大きく押し上げすぎない
- 2歳戦では、勝ちパターン分析、当日オッズ、馬場適性、馬体重の比重を相対的に上げる
- `best-time` が上位でも、少頭数・初期サンプル・極端な相手差の可能性を前提に過信しない
- `closing-speed` が良くても、脚質が不安定なら補正は小さめに留める

実務上の扱い:

- 2歳戦では `best-time` と `closing-speed` は通常戦の半分から 3分の2 程度の感覚で使う
- 2歳戦で本命を押し上げる主因は、時計系より `pattern_uma.track_condition_rates` と市場支持の重なりを優先する

## 当日開催傾向の使い方

`GET /nankan/meetings/{date}/{course}/races/{race_no}/trend-context` は、開催日その日の流れを見る補助材料として使うこと。
事前予想では `/trend` を直接採用せず、`trend-context.usable` が `true` のときだけ当日傾向として使う。

見る対象:

- 枠傾向
- 脚質傾向
- 騎手傾向
- 厩舎傾向

使い方の原則:

- ベース予想をひっくり返す主因にはしない
- 僅差の馬の上下や、相手候補の取捨に使う
- `race_count_completed` が少ないときほど軽く扱う
- 対象レースが `N` R の事前予想なら、`race_count_completed >= N` の trend は開催後時点の値として採用しない
- trend を不採用にした場合は、`使用データ` を `当日開催傾向: no` とし、注意点に `開催傾向は取得値が開催後時点のため今回は不採用` と書く
- `trend-context.usable` が `false` の場合は、`reason` を根拠に不採用理由を書く

反映の目安:

- `1R-3R終了時`:
  - 参考程度
  - 影響はかなり弱くする
- `4R-7R終了時`:
  - 軽い補正として使う
- `8R以降`:
  - 当日の流れとしてある程度信頼してよい

出力時に必ず書くこと:

- どの傾向を見たか
- `race_count_completed` がいくつか
- 予想全体の中でどのくらい見たか

表現の目安:

- `参考程度`
- `弱めに反映`
- `中程度に反映`

例:

- `当日傾向は6R終了時点なので弱めに反映`
- `前有利傾向を中程度に反映`
- `笹川翼の当日好調を相手評価に軽く加点`

## 熱さ判定の目安

`熱い！` と言ってよいのは、次のような重なりがあるときです。

- 本命候補がパターン評価で明確な1位
- 単勝または複勝オッズでも強い支持がある
- 当日馬場と `track_condition_rates` が噛み合う
- 馬体重増減が極端でない
- 相手候補も絞りやすく、買い目の形が素直

追加条件:

- 人気上位同士が接戦のときは `熱い！` を出さない
- 目安として、上位2頭の単勝オッズ差が小さい、または本命候補と対抗候補の根拠差が薄いときは `通常` に留める
- パターン・オッズ・馬場の3点が揃っていても、相手候補が横並びなら `熱い！` を避ける

逆に、次のようなときは `熱い！` を避けること。

- オッズとパターンが食い違う
- 馬場適性の根拠が薄い
- サンプル数が少ない高率に依存している
- 相手が広がりすぎて券種が散る
- 人気上位同士が接戦
- 本命候補と対抗候補の差が説明し切れない
