# SQL出力フォーマット規約

## 1. ファイル冒頭ヘッダーコメント（必須）

```sql
-- /////////////////////////////////////////////////////////////
-- チケット #【番号】 【件名】
-- 【集計・抽出の概要（1行）】
-- 対象期間: 【期間の説明】
-- 除外: 【除外条件（テーブル.カラム = 値 の形式）】
-- フィルタ: 【フィルタの説明】
-- 出典: 【DB名】 / 【主テーブル名】 実績ベース
-- 作成日: 【YYYY-MM-DD】
-- /////////////////////////////////////////////////////////////
```

## 2. DECLARE ブロック

- フィルタ変数は冒頭にまとめる
- 変数の直上に `-- ★` コメントで用途・変更方法を記載
- NULL = 全件対象のパターンを標準とする

```sql
-- ★ 〇〇フィルタ設定
-- NULL = 全件対象。特定に絞る場合は数値をセット（例: SET @xxx = 3）
DECLARE @xxx INT = NULL;

-- ★ 期間設定
DECLARE @date_from DATE = '2023-03-01';
DECLARE @date_to   DATE = '2026-02-28';
```

## 3. クエリブロックの区切り

各クエリ（メイン・補足・参考）の前に区切りコメントを入れる:

```sql
-- /////////////////////////////////////////////////////////////
-- メインクエリ: 【目的】
-- /////////////////////////////////////////////////////////////
```

補足クエリ・参考クエリの場合:
```sql
-- /////////////////////////////////////////////////////////////
-- 補足: 〇〇件数確認（〇〇のため事前に確認すること）
-- /////////////////////////////////////////////////////////////

-- /////////////////////////////////////////////////////////////
-- 参考: 〇〇マスタ確認（〇〇のIDを把握するために使用）
-- /////////////////////////////////////////////////////////////
```

## 4. SELECT 列のグループコメント

列のグループが変わる箇所に `-- グループ名` を挿入:

```sql
SELECT
    -- 会計期
    CASE ... END AS 会計期,
    -- 年月
    YEAR(o.created_at)  AS 注文年,
    MONTH(o.created_at) AS 注文月,
    -- 発送方法
    COALESCE(...) AS 発送方法,
    -- サプライヤー
    puo.supplier_id AS サプライヤーID,
    -- 件数集計
    COUNT(DISTINCT o.id) AS 注文件数,
```

## 5. FROM / JOIN のコメント

各 JOIN の直前に結合目的をコメント:

```sql
FROM idoz0804db.orders AS o

-- 注文明細
INNER JOIN idoz0804db.order_line_items AS oli
    ON o.id = oli.order_id

-- 商品マスタ（国内発送除外のため）
INNER JOIN idoz0804db.products AS p
    ON pu.product_id = p.id
```

## 6. WHERE 条件のコメント

各条件行の末尾または直上に日本語コメント:

```sql
WHERE
    -- 期間フィルタ
    o.created_at >= @date_from
    AND o.created_at < DATEADD(DAY, 1, @date_to)

    AND p.shipment <> 0   -- 国内発送商品を除外

    -- BigCategory フィルタ（NULL = 全件）
    AND (@big_category_id IS NULL OR bc.id = @big_category_id)
```

## 7. ★ マークの使い方

`-- ★` で始まるコメント = 「実行前に変更・確認が必要な箇所」:
- DECLARE 変数の直上
- WHERE 句の対象ID・期間指定
- 再利用時に変更が必要なハードコーディング値

## 8. 複数クエリの分割

- T-SQL: `GO` で区切る
- 補足クエリ・参考クエリは末尾に追加

## 9. 列エイリアスは日本語

SQLの出力列名は日本語エイリアスを付ける（Excelで開いた際に可読性を確保するため）。
