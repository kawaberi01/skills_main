# Dansou Keiba Reference

南関競馬の予想本体を差し替えず、最後に参考欄として短く構造化して添えるためのスキルです。

## 基本方針

- 予想順位、買い目、本命評価そのものはこのスキルで上書きしない
- 情報取得は必ず API から行う
- API に無い情報を推測の主材料にしない
- 出力は必ず固定テンプレートを使う
- 次の一手は 1 つだけにする

## 取得ルール

まず [references/api-materials.md](references/api-materials.md) を読むこと。

対象レースが指定されたら、原則として次を確認する。

1. `card`
2. `odds`
3. `pattern`
4. 必要に応じて `trend`
5. 必要に応じて `best-time` / `closing-speed` / `style-profile`

不足 API がある場合:

- 取得できた情報だけで簡潔に整理する
- 不足している API 名を `今やらないこと` または `思考コメント` で明示する

## このスキルでやること

断層競馬の参考欄として、次を短く整理する。

1. いまのレースをどういう作業として見るか
2. どの粒度で見るべきか
3. いま見るべき材料と、まだ踏み込まないこと
4. どの観点でレースを切るか
5. 次に取る 1 アクション

## 作業分類

レース補助コメントでは、次から 1 つ選ぶ。

- 概算（ざっくり判断で足りる）
- 調査（材料確認が中心）
- 設計（頭候補 / 軸候補 / 相手候補の組み立て）
- 実装（具体的な買い方へ落とし込む）
- 改善 / 最適化（買い方や絞り方の微修正）
- 不明

原則:

- 予想前の参考欄なら `調査` または `設計`
- 買い方まで明確に寄せる補助欄なら `実装`
- 振り返りや比較なら `改善 / 最適化`

## 粒度

次から 1 つ選ぶ。

- 粗い
- 中
- 詳細

原則:

- レース前半や材料不足時は `粗い` または `中`
- 直前で API 材料が揃っているときだけ `詳細`

## 候補手法

断層競馬の参考欄では、主に次の観点から 2 〜 3 個だけ選ぶ。

- 馬場一致
- 場・距離一致
- 騎手×厩舎一致
- 人気と条件のズレ
- 当日傾向との接続

## 固定テンプレート

出力テンプレートは [references/template.md](references/template.md) をそのまま使うこと。

見出し名、順番、書き方は崩さないこと。

## 禁止

- 長い説明
- 予想本体をこの欄で再説明すること
- 複数の `次の一手`
- API で未取得の情報を主根拠にすること
- 断層競馬の参考欄だけで本命や買い目を断定し直すこと
