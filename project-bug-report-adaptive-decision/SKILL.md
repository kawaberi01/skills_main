# Project Bug Report Adaptive Decision

調査結果を次の工程判断へ変換するスキル。

## 目的

- 追加確認を広げ過ぎない
- 誤修正リスクが高い状態で実装へ流さない
- 必要なときだけ限定確認を追加する
- 実装へ渡せるなら、その条件を短く固定する
- トークン上限に達した場合は、未確認理由を明示して停止する

## 判定区分

### 停止

- 対象プロジェクトが未確定
- 有力候補が複数残る
- 本番データや環境差分の確認なしに断定できない
- token guard の読み込み上限に達している

### 限定確認

- 有力候補はあるが、少数の影響先確認が必要
- 共通基盤、DB 定義、設定差分の局所確認だけで判断を前進できる

### 実装引き継ぎ

- 根本原因に十分なコード根拠がある
- 変更箇所が局所化できる
- 回帰確認観点が整理できている

## 手順

1. evidence の出力から、停止 / 限定確認 / 実装引き継ぎ を判定する
2. 限定確認が必要な場合だけ、確認対象を最大3項目に絞る
3. 実装引き継ぎ可能な場合だけ、最小修正単位と回帰観点を整理する
4. 工数、代替案比較、全件影響範囲は必要な根拠がない限り省略する
5. 読み込み上限に達した場合は、追加調査候補として記録する

## 出力

1. 判定結果
2. 判定根拠
3. 限定確認対象
4. 最小修正単位
5. 回帰確認観点
6. 推奨する次アクション
