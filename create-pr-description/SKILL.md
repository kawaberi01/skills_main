# PR説明文 自動生成スキル

ワークスペースのgit差分を分析し、構造化されたPR説明文をマークダウンで生成する。

## 前提条件
- PRのブランチがチェックアウト済みであること
- ワークスペースにgitリポジトリが存在すること

## ワークフロー

### Step 1: ブランチ情報と差分の取得

```bash
# 現在のブランチ名を確認
git branch --show-current

# リモートブランチの最新を取得
git fetch origin

# ベースブランチを特定（master or main）
git branch -r | grep -E "(origin/master|origin/main|origin/develop)"

# 分岐点のコミットIDを取得（ベースブランチに合わせて変更）
git merge-base HEAD origin/master

# コミットログを取得（分岐点から現在まで）
git log <分岐点コミットID>..HEAD --oneline

# 変更ファイル一覧を取得
git diff <分岐点コミットID>..HEAD --name-status

# 変更の統計情報
git diff <分岐点コミットID>..HEAD --stat

# 各ファイルの詳細差分を取得
git diff <分岐点コミットID>..HEAD
```

**注意**: 差分が大きい場合はファイルごとに分割して取得する。

### Step 2: 変更内容の分析

差分から以下を特定する：
1. **変更の目的** — コミットメッセージ・変更パターンから推測
2. **主要な変更ファイル** — 新規/修正/削除それぞれをカテゴリ分け
3. **構造的変更** — メソッド追加・抽出・リファクタリング等
4. **テストコード** — テスト追加/修正の有無と検証内容
5. **パッケージ・依存関係** — NuGet/npm等の追加・更新
6. **設定変更** — appsettings、config等の変更

### Step 3: 関連Redmineチケットの確認（任意）

ユーザーがRedmineチケット番号を指定した場合、または差分のコミットメッセージ・ブランチ名にチケット番号が含まれる場合：
1. Redmine MCPサーバーでチケット情報を取得
2. チケットの重要度・緊急度・期日を取得
3. PR説明文の「関連するRedmineチケット」セクションに反映

### Step 4: PR説明文の生成

[references/template.md](references/template.md) のテンプレートに従い、分析結果からPR説明文を生成する。

**生成のガイドライン**:
- **概要**: 1〜3文で変更の目的と影響を簡潔に記述
- **修正内容**: ファイル単位でグループ化し、変更の要点を箇条書き
  - ファイル名は太字で記載
  - 新規メソッド/クラスは明示的に記載
  - リファクタリングの場合は「何を」「どう変えたか」を記載
- **テストコード**: テストの種類（単体/統合）と検証項目を列挙
- **動作検証**: テストで確認できた動作を具体的に列記
- **関連するRedmineチケット**: URL・重要度・緊急度・期日を記載
- **その他**: 補足事項があれば記載（なければ空欄）

### Step 5: 出力

生成したPR説明文をユーザーに提示する。ユーザーが修正を求めた場合は反映する。

## プロジェクトとリポジトリの対応

Azure DevOpsのプロジェクト・リポジトリIDは [references/projects.md](references/projects.md) を参照。

## 注意事項
- すべての出力は**日本語**で記述する
- 推測が難しい場合は「概要」に仮の記述を入れ、ユーザーに確認を促す
- 差分の規模に応じて詳細度を調整する（小規模なら簡潔に、大規模なら詳細に）
- コード変更は一切行わない — PR説明文の生成のみ
