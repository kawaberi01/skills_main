# カスタムフィールド詳解（incident）

## 1. 部署/サイト (ID: 34) ⭐必須
- フィールドタイプ: リスト選択（ドロップダウン）
- 用途: チケットが属する事業部門/サイト部門の特定、ルーティング

**選択肢**
- iBeauty
- iDrug
- iRx
- iGeneric
- Mason
- 経営企画
- Ekimae
- 統括
- 開発部
- 店舗開発プロジェクト
- 輸出事業推進プロジェクト
- クリニック開発事業部

## 2. 重要度 (ID: 133) ⭐必須
- フィールドタイプ: リスト選択
- 基準: ビジネスインパクトと緊急性

| ランク | 定義 | 対象事例 |
|---|---|---|
| S：絶対にやる | 法改正、セキュリティ、致命的バグ、業務停止中 | システム障害、脆弱性、顧客影響の大きい不具合 |
| A：できればやる | 現場からの強い要望、業務効率が大きく改善 | 重要な要望、業務プロセス改善 |
| B：余裕あればやる | あったら便利だが急ぎでない | 利便性向上、細かい改善依頼 |

## 3. 緊急度 (ID: 134) ⭐必須
- フィールドタイプ: リスト選択
- 説明: 対応期限の目安

| 緊急度 | 対応期限 | 説明 | 対象事例 |
|---|---|---|---|
| 今日中 | 本日中 | 業務やサービスへの即座の影響 | システム障害、顧客への即時対応必須案件 |
| 3日以内 | 3営業日以内 | 対応遅延で業務支障の可能性 | 期限迫った依頼 |
| 1週間以内 | 翌週まで | 中程度の優先度 | 週次業務関連の準備 |
| 1ヶ月以内 | 月単位 | 中〜低優先度、計画対応 | 月次処理、ルーチン改善 |
| 6ヶ月以内 | 半期内 | 低優先度、対応漏れ防止記録 | 中長期改善提案、予算準備 |

## 4. 誰待ち (ID: 67)
- フィールドタイプ: ユーザー選択
- 必須: いいえ
- 用途: 「誰待ち」ステータスに使用（回答待ちの相手を明示）

## 5. 希望納期 (ID: 136)
- フィールドタイプ: 日付
- 必須: いいえ
- 用途: 完了希望日の指定（期日指定がある場合に記入）

## 6. 目標に含まれる (ID: 135) ⭐必須
- フィールドタイプ: 真偽値（はい/いいえ）
- 用途: KPI・目標達成度評価に含めるかの判定

**選択基準**
- はい: KPI/目標値に関連し成否が評価に直結
- いいえ: 付随業務、ルーチン、予期しない対応

## 7. 上長確認済み (ID: 166)
- フィールドタイプ: 真偽値（はい/いいえ）
- 必須: いいえ
- 用途: 報告者の上長から着手承認状況

## 8. 開発が必要 (ID: 199)
- フィールドタイプ: 真偽値（はい/いいえ）
- 必須: いいえ（対応メンバーのみ表示）
- 用途: 開発・デプロイ要否の判定、定期的な状況確認に利用

## カスタムフィールドでの絞り込み（重要）
Redmine API の一覧取得でフィルターする場合、`cf_<field_id>` 形式を使用します。

例：
```text
params: {
  project_id: "incident",
  cf_34: "iBeauty",
  cf_133: "A",
  cf_134: "今日中"
}
```
